# エージェント共通ルール

## 【最重要】作業前の必須手順

**すべてのエージェントは作業を始める前に、以下のファイルを必ず読むこと。**

```
company/shared/vision.md      ← ビジョン・方向性
company/shared/strategy.md    ← 現在の戦略・優先施策
company/shared/decisions.md   ← 意思決定ログ
company/shared/context.md     ← たくとの考え・背景情報
```

`shared/` が更新されていれば、それが現在の唯一の正しい情報源である。

---

## たくとからの指示（最優先）

1. **確認が必要な作業（2種類のみ）**
   - X への投稿内容（原稿をたくとに提示 → 承認後に投稿）
   - ページのビジュアル・デザイン・レイアウト変更（変更内容を提示 → 承認後に実装）

2. **確認不要・自律実行してよい作業** — 上記以外はすべて確認なしに実行する。

3. **並行作業可** — 担当範囲が重ならない限り、同時進行してよい。

4. **成果物の提示義務** — 完了したタスクは必ず `reports/` に記録し、たくとへ報告する。
   - ファイル名：`YYYY-MM-DD_担当者名.md`

5. **判断が難しい場合は即確認** — 方針変更・費用発生・想定外の問題は即時たくとへ報告。

6. **すべて日本語で** — 報告・原稿・コメント・ファイル内記述はすべて日本語。

7. **権限行使の原則**
   - たくとにしかできないこと（Search Console・X API・Vercel環境変数・外部サービス契約）はたくとに依頼する
   - ファイル操作・git commit/push・コード実装はエージェントが自律実行してよい

8. **セキュリティ最優先** — APIキー露出・入力バリデーション不備を発見したら即報告・即対処。

---

## 絶対順守事項

- **破壊的操作禁止** — ファイル削除・git reset --hard・既存コードの大幅削除は単独実行禁止。必ずたくとの承認を得る。
- **外部投稿の制限** — X・LINE・外部サービスへの実際の投稿は行わない。原稿作成までが職務範囲。
- **最終意思決定の委任** — 方針変更・新機能追加・費用発生・外部サービス契約はたくとの承認必須。

---

## 報告フォーマット（reports/に保存）

```
【実行者】名前（役職）
【実行日】YYYY-MM-DD
【タスク】何をしたか
【成果物】生成・変更したファイル一覧
【結果】成功 / 失敗 / 一部完了
【次のアクション】次にやるべきこと
【エスカレーション】たくとの判断が必要な事項（なければ「なし」）
```
