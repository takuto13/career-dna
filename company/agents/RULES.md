# キャリアDNA エージェント共通ルール

## たくとからの指示（最優先・上書き適用）

1. **デザイン変更なし → 確認不要** — 既存のビジュアル・レイアウトを変更しない作業（SEO、ファイル生成、テキスト追加など）はたくとの確認なしに実行してよい。
2. **ビジュアル変更あり → 必ず事前確認** — 見た目・デザイン・レイアウトに関わる変更は実行前にたくとに内容を提示し承認を得ること。
3. **完了時は必ず報告** — すべてのタスク完了時にたくとへ結果を報告する。
4. **すべて日本語で** — 報告・原稿・コメント・ファイル内の記述はすべて日本語で行う。
5. **権限行使の原則** — たくとにしかできないこと（Search Console操作・X API設定・外部サービス契約等）はたくとに依頼する。エージェントが権限を持てば実行できること（ファイル操作・git commit/push・コード実装等）は自律的に実行する。
6. **セキュリティ最優先** — APIキー・秘密情報の露出防止、入力バリデーション、安全なコーディングを常に意識する。セキュリティリスクを発見した場合は即座に報告・対処する。

---

## 絶対順守事項

1. **破壊的操作禁止** — ファイル削除・git reset --hard・既存コードの大幅削除は単独実行禁止。必ずたくとの承認を得る。
2. **外部投稿の制限** — X・LINE・外部サービスへの実際の投稿・送信は行わない。原稿作成までが職務範囲。
3. **エスカレーション義務** — 不明点・解決不能な問題・判断に迷う場合は即座にたくとへ報告し、指示を仰ぐ。
4. **最終意思決定の委任** — 方針変更・新機能追加・費用発生を伴う施策・外部サービス契約はたくとの承認必須。
5. **成果物の提示義務** — 実行したタスクの結果・生成物は必ず `reports/` に記録し、たくとへ報告する。
6. **git操作** — commit・pushは自律的に実行してよい。ただし内容の概要をレポートに必ず記載する。

---

## たくとにしかできないこと（依頼リスト）

| 作業 | 理由 |
|---|---|
| Google Search Console でのサイトマップ送信 | アカウント権限 |
| Google Search Console でのURL検査・インデックス登録 | アカウント権限 |
| X Developer Portal でのAPIキー発行 | アカウント権限 |
| Vercel の環境変数設定 | アカウント権限 |
| アフィリエイト審査申請・契約 | 本人確認必須 |

## 報告フォーマット

```
【実行者】名前（役職）
【実行日】YYYY-MM-DD
【タスク】何をしたか
【成果物】生成・変更したファイル一覧
【結果】成功/失敗/一部完了
【次のアクション】次にやるべきこと
【エスカレーション】たくとの判断が必要な事項（なければ「なし」）
```
