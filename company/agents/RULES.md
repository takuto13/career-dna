# エージェント共通ルール

## 【最重要】作業前の必須手順

すべてのエージェントは作業を始める前に、以下のファイルを必ずこの順番で読むこと。

```
company/shared/handoff.md        ← 前回の続きがあればここから開始（最優先）
company/shared/strategy.md       ← 現在の方針・やること/やらないこと
company/shared/decisions.md      ← 意思決定ログ（最新の方針変更はここに記録される）
company/BACKLOG.md               ← 今日やるべきタスク
company/shared/context.md        ← プロジェクト背景・現状
```

ファイルを読まずに作業を始めてはならない。

---

## 報告ルール（最重要）

- **報告書はCEOが1日1本だけ作成する**（`company/reports/YYYY-MM-DD.md`）
- CTO・SNS担当は個別の報告書を作成しない
- CTOは作業完了後にBACKLOGの該当タスクを `[x]` に更新する
- SNS担当は作業完了後にBACKLOGの該当タスクを `[x]` に更新し、投稿原稿を `company/x-posts/YYYY-MM-DD_原稿.md` に保存する
- CEO報告書にはCTO・SNSの実行内容・X投稿原稿・エスカレーション事項をすべて含める

---

## たくとの承認が必要なもの

| 種別 | 内容 |
|------|------|
| X投稿 | 投稿前にたくとが原稿を確認する（CEOの日次報告書で確認） |
| 難しいと判断した案件 | CTOが実装困難・リスク大と判断した変更（CEO報告書に記載してたくとへ確認） |

## CTOの実行権限

**流入増加につながる施策はバンバン実行してよい。** 迷ったら実行する。

### 自律実行してよい（たくとの承認不要）

- SEO・LLMO・構造化データ・サイトマップ・robots.txtの改善
- 流入に関わるテキスト修正（SEOキーワード追加・ページタイトル・meta・見出し）
- 内部リンクの追加・改善
- パフォーマンス改善（表示速度・画像最適化）
- GA4計測の追加・修正
- デザインに影響しない範囲でのHTML/CSS/JS変更

### たくとへエスカレーション（報告書に記載・実装しない）

- たくとにしかできない操作（Search Console・Vercel環境変数・外部サービス契約・X投稿）
- デザイン・レイアウトが変わる変更
- UI・遷移・CVR最適化の変更（流入データが取れるまで禁止）
- 破壊的な変更（大幅なコード削除・リファクタ）
- 実装方針の判断が難しい案件

## SNS担当の日次調査タスク

SNS担当は毎日、投稿原稿作成とは別に以下の調査を実施し、`company/shared/sns-intel.md` を更新する。

- X（Twitter）での転職・就活関連のトレンドワード・バズ投稿の把握
- 競合・同業アカウントの投稿傾向の確認
- 効果的なフック・構成パターンの発見

調査結果は全エージェントが次回作業前に参照する。

## トークン使用方針

- 1回の作業セッションでトークン使用率80%を目安に使い切ってよい。遠慮せず作業量を最大化する。
- 上限に近づいて作業が途中になる場合は、必ず `company/shared/handoff.md` に以下を記録してからセッションを終える。
  - 何をどこまでやったか
  - 次に何をやるか（具体的なタスクと手順）
  - 未完了の作業の現在状態
- 次のセッションは handoff.md を最初に読んで、続きから作業を開始する。
- handoff.md は続きの作業が完了したらその内容を削除（または「なし」に更新）する。

## 絶対順守事項

- **GitHub Issue確認は毎回必須** — スケジュール実行・たくとからの作業指示・いかなるタイミングでも、作業開始前に必ずIssue一覧を確認する。ownerラベルの有無にかかわらず全Issueを確認すること。確認ツール：`mcp__github__list_issues`
- **FAQ作成はたくとの明示的指示があるまで禁止** — faq.htmlおよび全タイプページへのFAQ追加は、たくとから「作成してよい」と明示されるまで行わない
- **git push は自律実行してよい** — 実装後は承認なしにそのままpushする。報告書に実行内容を記載すれば十分。ただしサイトの主要機能が壊れる・デザインが変わる変更の場合は事前にエスカレーションする。
- **破壊的操作禁止** — ファイル削除・git reset --hard・既存コードの大幅削除は単独実行禁止
- **外部投稿の制限** — X・外部サービスへの実際の投稿は行わない（原稿作成・保存まで）
- **UI・デザイン変更の禁止** — 流入データが取れるまで、見た目・遷移・CVR最適化の変更は行わない
- **すべて日本語で** — 報告・原稿・コメント・ファイル内記述はすべて日本語
- **方針変更・費用発生・想定外の問題** — 即時CEO報告書でたくとへエスカレーション
