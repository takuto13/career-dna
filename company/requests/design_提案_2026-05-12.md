# デザイン提案書 2026-05-12

**提案者：** CTO  
**対象ページ：** result.html（診断結果ページ）  
**目的：** アフィリエイトCTR最大化 / コンバージョン導線の最短化

---

## 背景・課題

現在の result.html のフロー：
1. 診断完了 → result.html で結果表示
2. 「このタイプに合う求人を見る →」ボタン（1個）をクリック
3. jobs.html へ遷移 → そこで広告カードを表示

**問題点：**
- 診断完了の高揚感がある「ゴールデンモーメント」にアフィリエイトリンクを直接見せられていない
- 2ステップ必要なためCTRが下がる
- 競合の「すべらない転職」「パーソナルファイル」は診断結果直下に「あなたにおすすめのエージェント」を表示しCTRを最大化している

---

## 提案内容

### 変更箇所：result.html の `.button-area` セクション直後

現在：
```
[このタイプに合う求人を見る →]  ← jobs.htmlへのリンク
[𝕏 結果をシェアする]
```

提案：
```
[このタイプに合う求人を見る →]
[𝕏 結果をシェアする]

━━━━━━━━━━━━━━━━━━━━━━━━━━━
▼ あなたの【タイプ名】に合うサービス TOP3
━━━━━━━━━━━━━━━━━━━━━━━━━━━

[サービスカード1（PRバッジ付き）]
 - サービス名・バッジ（SaaS特化・未経験OKなど）
 - 1行説明文
 - ポイント3点（✅ 形式）
 - 対象者注意書き
 - [無料で登録する →]  ← アフィリエイトリンク（ボタン形式）

[サービスカード2]
[サービスカード3]

 ↓ もっと見る → jobs.html
```

### デザイン仕様案

- カード背景：白 / 角丸16px / 影あり（既存の .result-card と同じスタイル）
- アフィリエイトCTAボタン：グラデーション（既存の .btn スタイル流用）
- PR表示：各カード右上に「PR」バッジ（法令遵守）
- モバイル：1カラム積み重ね表示

### 実装コスト

- result.html: `showResult()` 関数に広告カード生成ロジックを追加
- jobs.html: 変更なし（詳細ページとして残す）
- affiliate-config.js: 変更なし（既存データを流用）
- 追加CSS: `.result-affiliate-section`（約30行）

---

## 期待効果

- アフィリエイトCTR：現状比 **+30〜50%** 見込み（診断完了直後の高関心状態で接触）
- 1クリックでサービス登録ページへ到達（jobs.html経由をスキップ可能）
