### タスク1：職人・挑戦者タイプページ「転職成功ポイント」コンテンツセクション追加【最優先】

**背景・理由**
type-craftsman.html（職人タイプ）・type-challenger.html（挑戦者タイプ）は、それぞれ「コツコツできる仕事 転職」「行動力がある人 向いてる仕事」という競合多数のロングテールKWを狙っている。現状はFAQ・アフィリエイトカード・ジョブグリッドで構成されているが、コンテンツの「深さ」が競合記事より薄い可能性がある。実体験に近い Tips 型のテキストセクションを追加することで：
1. 記事の文字数・コンテンツ密度が増加 → Googleのコンテンツ品質評価向上
2. ロングテールKWの自然な追加挿入
3. ユーザーの滞在時間延長（読む価値のあるコンテンツ追加）

**対象ファイル：** `public/type-craftsman.html`・`public/type-challenger.html`

**追加するセクション（両ページ共通フォーマット）**

FAQセクションの「前」に、以下の内容のセクションを追加すること（既存のデザインと合わせて最小限のHTML）：

```
■ セクションタイトル（H2）：「[タイプ名]タイプが転職で成功するポイント」

■ 構成（300〜500文字目安の本文）：
- このタイプの転職活動の強みを1〜2文で説明
- 向いている転職先の職種・環境を具体的に2〜3例示
- 転職で失敗しやすいパターン（注意点）を1〜2文
- 「このタイプはキャリア診断で強みを確認してから動くと効果的」→ 診断TOP（/）への内部リンク
```

**職人タイプ（type-craftsman.html）向け内容ヒント：**
- 強み：専門スキルを深める仕事・コツコツ取り組む能力が企業に高く評価される
- 向く転職先：IT系（バックエンドエンジニア・データアナリスト）・製造業・会計・研究職
- 注意点：面接での「自己PR」が苦手なケースが多く、スキルの棚卸しが事前に必須
- ターゲットKW自然挿入：「コツコツできる仕事 転職」「専門職 向いてる人」

**挑戦者タイプ（type-challenger.html）向け内容ヒント：**
- 強み：高い目標設定・行動力・変化への適応力が新規事業・営業・スタートアップ環境で発揮
- 向く転職先：外資系・ベンチャー・営業職・プロジェクトマネージャー・起業支援職
- 注意点：安定志向の職場環境でモチベーションが下がりやすい。企業文化の確認が重要
- ターゲットKW自然挿入：「行動力がある人 向いてる仕事」「挑戦者 転職 おすすめ」

**実装方針**
- 既存のHTML構造・CSSクラスを確認してから、既存セクションのスタイルを流用すること
- 新規CSSクラスの追加は最小限にとどめること
- JSON-LDへの追加は不要（HTMLコンテンツのみ）
- sitemap.xml の type-craftsman.html・type-challenger.html の lastmod を 2026-05-20 に更新

**期待成果物**
- type-craftsman.html に「転職成功ポイント」セクション追加済み
- type-challenger.html に「転職成功ポイント」セクション追加済み
- sitemap.xml lastmod 更新済み
- コミット・プッシュ完了

**注意事項**
- **デザインが大きく変わる場合はたくとへエスカレーションして実装しない**こと
- 既存ページのレイアウト・カラーテーマを変更しないこと
- 文字数は目安300〜500文字。過度に長くしない

### タスク2：index.html WebSite JSON-LD（SearchAction）追加

**背景・理由**
Googleのサイトリンク検索ボックス（Sitelinks Searchbox）を促進するため、`WebSite` スキーマに `potentialAction` として `SearchAction` を追加する。これにより：
1. Googleの検索結果に「サイト内検索ボックス」が表示される可能性（キャリアDNAはタイプ名での検索がある）
2. Googleによるサイトのエンティティ認識強化
3. 実装コスト：10分以下・費用ゼロ・デメリットなし

**対象ファイル：** `public/index.html`

**実装内容**

index.htmlの既存JSON-LDブロックの後に、以下を追加：

```json
{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "name": "キャリアDNA",
  "url": "https://career-dna.jp",
  "potentialAction": {
    "@type": "SearchAction",
    "target": {
      "@type": "EntryPoint",
      "urlTemplate": "https://career-dna.jp/?q={search_term_string}"
    },
    "query-input": "required name=search_term_string"
  }
}
```

※ キャリアDNAにサイト内検索機能は現状ないが、スキーマの追加自体はGoogleに対してエンティティ情報を提供するため有効。ペナルティリスクなし。

**期待成果物**
- index.htmlにWebSite JSON-LD追加済み
- コミット・プッシュ完了

### タスク3：全タイプページ・主要ページの画像alt属性SEO最適化チェック

**背景・理由**
画像の `alt` 属性はGoogleの画像検索流入・アクセシビリティ・E-E-A-T評価に影響する。各ページで `alt=""` や未設定の画像がある場合は、適切なキーワードを含むalt属性に更新する。

**対象ファイル：** `public/type-*.html`（全8タイプ）・`public/index.html`

**確認・実施内容**
1. 各ファイルの `<img>` タグを確認
2. `alt=""` または `alt` 属性なしの画像に適切なテキストを設定
   - 例：タイプアイコン → `alt="職人タイプのアイコン - コツコツ型の仕事タイプ"`
   - 例：ロゴ・ブランド画像 → `alt="キャリアDNA - AIキャリア診断"`
3. 既にSEO的に適切なalt属性がある場合は変更不要

**期待成果物**
- 全タイプページのalt属性確認・更新完了（または「問題なし」の確認）
- コミット・プッシュ（変更があった場合のみ）
