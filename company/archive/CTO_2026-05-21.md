### タスク1：守護者・育成者・調整役タイプページ「転職成功ポイント」セクション追加【最優先】

**背景・理由**
type-craftsman・type-challengerへの実装（05-20）に続き、守護者・育成者・調整役の3ページにも同形式のコンテンツセクションを追加する。対象KWは競合記事が多いが「診断体験との連動」がない記事ばかりで差別化可能。

**対象ファイル：**
- `public/type-guardian.html`（守護者タイプ）
- `public/type-nurturer.html`（育成者タイプ）
- `public/type-mediator.html`（調整役タイプ）

**追加するセクション（3ページ共通フォーマット）**

FAQセクションの「前」に以下を追加すること。既存ページのHTMLクラス・スタイルを流用し、新規CSSは最小限にとどめること。

```
■ セクションタイトル（H2）：「[タイプ名]タイプが転職で成功するポイント」
■ 構成（300〜500文字目安）：
- このタイプの転職活動の強みを1〜2文
- 向いている転職先の職種・環境を具体的に2〜3例示
- 転職で失敗しやすいパターン（注意点）を1〜2文
- 診断TOP（/）への内部リンク
```

**各タイプの内容ヒント：**

**守護者タイプ（type-guardian.html）**
- KWターゲット：「責任感が強い人 向いてる仕事」「真面目な人 向いている職種」
- 強み：責任感・安定志向・チームへの献身性が組織から高く評価される
- 向く転職先：公務員・医療・インフラ・総務・品質管理・法務
- 注意点：「変化の少ない環境」を選びがちだが、成長機会が限られる場合も。仕事の「安心感の源泉」を事前に言語化することが重要
- 内部リンク文言：「守護者タイプの強みをより詳しく知りたい方は、AIキャリア診断で確認しましょう」

**育成者タイプ（type-nurturer.html）**
- KWターゲット：「人を育てる仕事 向いている職種」「人の成長を支援する仕事」
- 強み：傾聴力・共感力・サポート力が教育・医療・HR系で圧倒的に活かされる
- 向く転職先：教育・研修講師・人事・キャリアアドバイザー・社会福祉・看護
- 注意点：「他者優先」が強すぎて自分のキャリアが後回しになりがち。強みを明確に言語化してから転職活動に臨むこと
- 内部リンク文言：「育成者タイプの強みをAIキャリア診断で確認してから動き出そう」

**調整役タイプ（type-mediator.html）**
- KWターゲット：「コミュニケーション力 向いている仕事 転職」「調整役 得意な人 職種」
- 強み：利害調整・チームビルディング・橋渡し役として、組織内外で不可欠な存在になれる
- 向く転職先：PM（プロジェクトマネージャー）・営業・HR・コンサル・広報・カスタマーサクセス
- 注意点：「自分の意見」を主張するのが苦手で、交渉や給与交渉で損をしやすい。転職では「強みの棚卸し」が特に重要
- 内部リンク文言：「調整役タイプの特徴と向いている仕事は、AIキャリア診断で詳しく確認できます」

**期待成果物**
- type-guardian.html に「転職成功ポイント」セクション追加済み
- type-nurturer.html に「転職成功ポイント」セクション追加済み
- type-mediator.html に「転職成功ポイント」セクション追加済み
- sitemap.xml の対象3ページ lastmod → 2026-05-21 に更新
- コミット・プッシュ完了

**注意事項**
- デザインが大きく変わる場合はたくとへエスカレーション・実装しない
- 既存ページのレイアウト・カラーテーマを変更しない
- 文字数は目安300〜500文字

### タスク2：jobs.html「タイプ別エージェント選び方」コンテンツセクション追加

**背景・理由**
jobs.htmlは現在「タイプ別おすすめサービス一覧」のリスト表示のみ。「転職エージェント 選び方 タイプ」「タイプ別 転職エージェント おすすめ」のロングテールKWに対応するコンテンツが存在しない。SEO流入獲得と同時にアフィリエイト誘導の文脈を強化できる、収益化直結の施策。

**対象ファイル：** `public/jobs.html`

**追加するセクション**

現在のサービス一覧の「上」に以下のセクションを追加すること：

```
■ セクションタイトル（H2）：「あなたのキャリアタイプに合ったエージェントの選び方」

■ 構成（400〜600文字目安）：
- 「転職エージェントは種類が多く、タイプによって相性が異なります」→ 読者に刺さる導入
- タイプ別の選び方ポイント（全タイプを網羅する必要はない。4〜6タイプのエッセンスを箇条書きまたはシンプルなリストで）
  例：「行動力が高い挑戦者タイプ → ハイクラス・外資系強みのエージェントと相性◎」
      「コツコツ型の職人タイプ → 専門職特化型・IT系エージェントが向いている」
      「人を支援する育成者タイプ → 教育・医療・HR特化のエージェントがマッチしやすい」
- 「まず自分のタイプを確認してから選ぶと、エージェントとの初回面談もスムーズ」
- 診断TOP（/）への内部リンクCTA
```

**実装方針**
- 既存のHTML構造・CSSクラスを確認してから流用すること
- 新規CSSクラスは最小限
- このセクションのFAQPage JSON-LD追加は不要（HTMLコンテンツのみ）
- sitemap.xml の jobs.html lastmod → 2026-05-21 に更新

**期待成果物**
- jobs.html に「タイプ別エージェント選び方」セクション追加済み
- sitemap.xml jobs.html lastmod 更新済み
- コミット・プッシュ完了

**注意事項**
- デザインが大きく変わる場合はたくとへエスカレーション・実装しない
- 既存のサービスカード一覧は変更しないこと
- 「このエージェントが最強」等の断言表現は避ける（法的リスク）
