## 【タスク1・最優先】全タイプページに BreadcrumbList JSON-LD を追加
**優先度：高 / 推定所要時間：30分**

### 背景・理由
全8タイプページ（type-craftsman / type-strategist / type-mediator / type-challenger / type-analyst / type-creator / type-guardian / type-nurturer）に BreadcrumbList の構造化データがない。追加することで：
- Googleの検索結果にパンくずリストが表示され、CTRが向上する
- AI検索エンジン（Perplexity等）がページの文脈を正確に理解しやすくなる（LLMO対策）

### 実装内容
各タイプページの `<head>` 内の `</head>` 直前に以下の JSON-LD を追加（既存 JSON-LD とは別の `<script>` タグ）：

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BreadcrumbList",
  "itemListElement": [
    {
      "@type": "ListItem",
      "position": 1,
      "name": "キャリアDNA",
      "item": "https://career-dna.jp/"
    },
    {
      "@type": "ListItem",
      "position": 2,
      "name": "○○タイプ詳細",
      "item": "https://career-dna.jp/type-○○.html"
    }
  ]
}
</script>
```

各ページの `name` と `item` は以下に合わせること：

| ファイル | name | item |
|---------|------|------|
| type-craftsman.html | 職人タイプ詳細 | https://career-dna.jp/type-craftsman.html |
| type-strategist.html | 戦略家タイプ詳細 | https://career-dna.jp/type-strategist.html |
| type-mediator.html | 調整役タイプ詳細 | https://career-dna.jp/type-mediator.html |
| type-challenger.html | 挑戦者タイプ詳細 | https://career-dna.jp/type-challenger.html |
| type-analyst.html | 分析者タイプ詳細 | https://career-dna.jp/type-analyst.html |
| type-creator.html | 表現者タイプ詳細 | https://career-dna.jp/type-creator.html |
| type-guardian.html | 守護者タイプ詳細 | https://career-dna.jp/type-guardian.html |
| type-nurturer.html | 育成者タイプ詳細 | https://career-dna.jp/type-nurturer.html |

## 【タスク2・高優先】vercel.json にリダイレクト設定を追加
**優先度：高 / 推定所要時間：15分**

### 背景・理由
- type-strategy.html は旧バージョンの戦略家タイプページ
- canonical と robots.txt Disallow は対応済みだが、ブラウザ・Googlebotが旧URLにアクセスした場合の301リダイレクトが未設定
- vercel.json で設定することで重複コンテンツ問題を完全解消できる

### 実装内容
プロジェクトルートに `vercel.json` が存在する場合は既存設定を保持して `redirects` を追加。存在しない場合は新規作成。

```json
{
  "redirects": [
    {
      "source": "/type-strategy.html",
      "destination": "/type-strategist.html",
      "permanent": true
    }
  ]
}
```

既存の `vercel.json` に他の設定がある場合は、それを壊さずに `redirects` キーを追加すること。

### 期待成果物
- vercel.json の作成 or 更新
- git commit & push

## 【タスク3・中優先】sitemap.xml の priority・changefreq を追加・最適化
**優先度：中 / 推定所要時間：20分**

### 背景・理由
Googleに対してクロール優先度を明示し、インデックス速度を上げる。

### 実装内容
sitemap.xml の各 URL に以下の値を設定：

| ページ種別 | priority | changefreq |
|-----------|----------|------------|
| index.html（診断ページ） | 1.0 | weekly |
| タイプ別詳細ページ 8ページ | 0.8 | monthly |
| faq.html | 0.7 | monthly |
| about.html | 0.6 | monthly |
| jobs.html（求人・アフィリエイトページ） | 0.7 | weekly |

### 期待成果物

- git commit & push
